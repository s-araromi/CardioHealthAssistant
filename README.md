# CardioHealthAssistant

CardioHealthAssistant is an AI-powered cardiovascular health companion that assists users in tracking, managing, and improving their cardiovascular health. Built with the Gemini API and Google AppSheet, this tool provides an interactive experience for users to set reminders, monitor health metrics, and receive evidence-based recommendations for heart health.

## Project Overview

CardioHealthAssistant is designed to support users in managing and optimizing their cardiovascular health through real-time, AI-driven guidance. It offers an all-in-one solution for promoting cardiovascular health by tracking metrics, setting reminders, and delivering tailored advice. Core capabilities include:
- Monitoring blood pressure, heart rate, and other cardiovascular health metrics
- Personalized insights and reminders for medication adherence and checkups
- Evidence-based exercise, diet, and stress management recommendations
- Emergency alert notifications for critical health conditions

This assistant respects privacy and ensures all user data is managed locally.

---

## Features

- **Health Metric Monitoring**: Track cardiovascular health metrics like blood pressure and heart rate.
- **Personalized Recommendations**: Suggests exercise, diet, and lifestyle changes based on cardiovascular health.
- **Reminders**: Customizable reminders for medication, health checkups, and regular monitoring.
- **Emergency Detection**: Alerts users to critical health conditions based on their data inputs.
- **User-Friendly Interface**: Leverages Google AppSheet for a simple user experience on web and mobile.

## Setup Instructions

### Requirements
- Python 3.7 or higher
- [Gemini API Key](https://example.com) (from Google AI Studio)
- Google AppSheet account for app deployment (if using mobile functionality)

### Installation

1. **Clone the Repository**
    ```bash
    git clone https://github.com/username/CardioHealthAssistant.git
    cd CardioHealthAssistant
    ```

2. **Create a Virtual Environment**
    ```bash
    python3 -m venv venv
    source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
    ```

3. **Install Dependencies**
    ```bash
    pip install -r requirements.txt
    ```

4. **Environment Variables**
   - Create a `.env` file in the root directory and add your Gemini API key:
     ```
     GEMINI_API_KEY=your_api_key_here
     ```

5. **Create a .gitignore file** (optional)
    To exclude unnecessary files:
    ```
    __pycache__/
    venv/
    *.log
    .env
    ```

### Running the Application

1. **Initialize and Run the Application**
   ```bash
   streamlit run app.py
   ```

2. Open the app in your browser (typically at `http://localhost:8501`).

## Usage Guide

## Technical Details

- **Language Model**: Uses the Gemini API for AI-driven responses.
- **Logging**: Logs interactions in `chatbot.log` for tracking and troubleshooting.
- **Session Management**: Utilizes Streamlit session state for chat history and user data.

### Key Files

- **app.py**: Streamlit application file for the web interface.
- **chat.py**: Contains logic for Gemini API interactions.
- **chatbot.log**: Log file for tracking interactions.
- **venv/**: Virtual environment for dependencies.
- **__pycache__/**: Temporary folder generated by Python for module caching.

## Troubleshooting

- **API Key Errors**: Verify that your Gemini API key is valid and correctly stored in the `.env` file.
- **Logging Issues**: Use `chatbot.log` to track errors or unusual responses.
- **Session State Errors**: Restart the app if `st.session_state` issues occur and check that all variables are initialized in `app.py`.
